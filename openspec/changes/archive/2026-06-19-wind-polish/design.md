## Context

Wind 项目已有基础框架，需要完善功能和修复问题。当前状态：
- 文件树基本可用但有交互问题
- 终端面板基础功能完成
- 编辑器面板只有欢迎页面
- 缺少文件预览功能

## Goals / Non-Goals

**Goals:**
- 修复所有 UI 交互问题
- 实现文件预览（语法高亮、Markdown）
- 完善面板布局（可调整大小）
- 实现基本的命令面板

**Non-Goals:**
- 不实现完整的 Neovim 集成（太复杂）
- 不实现所有文件格式预览（先做核心的）
- 不实现插件系统

## Decisions

### D1: 优先修复交互问题

**选择**: 先确保基本功能可用，再添加新功能

**理由**: 用户体验优先

### D2: 文件预览使用 Shiki

**选择**: 使用已安装的 Shiki 库实现语法高亮

**理由**: Shiki 已经是依赖，支持 200+ 语言

### D3: 面板调整使用 CSS Resize

**选择**: 使用 CSS resize 和鼠标拖拽实现面板大小调整

**理由**: 简单高效，无需额外依赖

## Risks / Trade-offs

**R1: 交互修复可能引入新问题**
→ 缓解: 逐步修复，充分测试

**R2: 文件预览性能**
→ 缓解: 延迟加载，大文件使用虚拟滚动
