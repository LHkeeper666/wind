## Context

Wind 是一个轻量级 Windows 文件工作台，当前已实现基础的 ImagePreviewer，支持 png/jpg/gif/svg/webp/bmp/ico 格式的显示。但现有实现只提供了静态展示，缺少交互能力和大图像优化。

技术决策文档中明确要求：
- 图像预览器使用原生 `<img>` + 缩放平移
- 图片 > 5000px 时降采样显示，提示原图尺寸

## Goals / Non-Goals

**Goals:**
- 实现图像缩放功能（鼠标滚轮、双击、按钮）
- 实现图像平移功能（鼠标拖拽）
- 处理大图像降采样（>5000px）
- 显示原图尺寸信息
- 优化内存管理

**Non-Goals:**
- 不实现图像编辑功能（裁剪、旋转、滤镜）
- 不支持多图像幻灯片浏览
- 不实现图像元数据（EXIF）显示

## Decisions

### 1. 缩放平移实现方案

**选择**: CSS transform + JavaScript 事件处理

**替代方案**:
- Canvas 渲染：性能好但实现复杂，需要重绘
- SVG viewBox：适合矢量图，不适合位图
- CSS transform：简单高效，浏览器原生支持

**理由**:
- CSS transform 的 `scale()` 和 `translate()` 性能好
- 浏览器会自动优化，使用 GPU 加热
- 实现简单，代码量少

**实现细节**:
```typescript
// 状态管理
let scale = 1;
let translateX = 0;
let translateY = 0;

// 应用变换
function applyTransform() {
  img.style.transform = `translate(${translateX}px, ${translateY}px) scale(${scale})`;
}
```

### 2. 缩放控制方式

**选择**: 多种缩放方式组合

- **鼠标滚轮**: `Ctrl + 滚轮` 缩放，避免与页面滚动冲突
- **双击缩放**: 双击切换 100% 和适应窗口
- **按钮控制**: 提供 +/- 按钮和重置按钮
- **键盘快捷键**: `+`/`-` 缩放，`0` 重置

**理由**: 提供多种交互方式，适应不同用户习惯

### 3. 平移控制方式

**选择**: 鼠标拖拽平移

**实现细节**:
- `mousedown` 记录起始位置
- `mousemove` 计算偏移并应用
- `mouseup` 结束拖拽
- 边界检查：防止图像完全拖出视口

### 4. 大图像处理策略

**选择**: Canvas 降采样 + 尺寸提示

**处理流程**:
1. 检查图像尺寸（宽度或高度 > 5000px）
2. 使用 Canvas 进行降采样（保持宽高比）
3. 降采样到 2000px 以内
4. 显示原图尺寸和降采样提示
5. 提供"查看原图"按钮（可选）

**理由**:
- 避免浏览器内存溢出
- 保持预览流畅性
- 用户知道这是降采样版本

### 5. 内存管理策略

**选择**: 及时释放 Object URL

**实现细节**:
- `dispose()` 时调用 `URL.revokeObjectURL()`
- 图片加载完成后保留 URL 引用
- 避免内存泄漏

## Risks / Trade-offs

**[风险] 大图像降采样可能丢失细节**
→ 缓解：提供"查看原图"选项，用户可选择加载原图

**[风险] CSS transform 在极端缩放下可能模糊**
→ 缓解：限制缩放范围（0.1x - 10x），超过范围时提示

**[风险] 触摸板缩放可能与鼠标滚轮冲突**
→ 缓解：统一使用 `wheel` 事件，通过 `deltaY` 判断缩放方向

**[权衡] 降采样会增加初始加载时间**
→ 接受：为了流畅的预览体验，值得额外的处理时间
